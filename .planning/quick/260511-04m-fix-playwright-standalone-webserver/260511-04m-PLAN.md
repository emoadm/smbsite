<objective>
Fix Playwright CI webServer so it actually boots the app under Next.js `output: 'standalone'`.

`next.config.ts` sets `output: 'standalone'` (intentional, for Fly.io Docker image — see CLAUDE.md "Application hosting"). Next.js then refuses to serve via `next start` and prints:

> "next start" does not work with "output: standalone" configuration. Use "node .next/standalone/server.js" instead.

Result on PR #2: nothing listens on :3000, Playwright waits 120s, ~37 specs fail with page-load timeouts (landing.spec, branding.spec, unsubscribe.spec, etc.).

The standalone build also does NOT copy `.next/static/` or `public/` into `.next/standalone/`. Those must be copied manually before booting `server.js`, otherwise CSS/JS/fonts/og-image/etc. 404 and the page renders blank.

**Fix:** Introduce a single `pnpm start:standalone` script that does the copy + exec, and point Playwright's CI webServer command at it. Chosen over (A) inlining a multi-line shell in `playwright.config.ts` and (C) splitting the copy into a separate CI step, because:

- Identical behaviour in CI and in local prod-equivalent runs (`CI=1 pnpm exec playwright test`) — option (C) breaks local CI-mode.
- Keeps `playwright.config.ts` readable — option (A) buries the command logic in TS string.
- Matches the existing CLAUDE.md convention of pnpm scripts for repeatable commands.

Purpose: unblock CI smoke run on PR #2 by closing the standalone webServer bug. The 3 Phase 5 env-var-assert failures (admin-newsletter-composer, community-page, newsletter-preferences) remain — separate bug, out of scope.

Output: Working `pnpm start:standalone` script and updated `playwright.config.ts` so CI Playwright job boots :3000 within timeout.
</objective>

  <action>
**Step 1 — Edit `package.json` scripts.**

Insert a `start:standalone` script immediately after the existing `"start": "next start",` entry. Use a single-line shell command (npm-script-friendly, works cross-platform via sh on Linux/macOS — CI runner is `ubuntu-latest` per ci.yml line 8, so POSIX `sh` is guaranteed):

```json
"start:standalone": "cp -r .next/static .next/standalone/.next/static && cp -r public .next/standalone/public && node .next/standalone/server.js",
```

Notes:
- The two `cp -r` calls populate the static-asset directories that Next.js's standalone build deliberately omits (documented behaviour — only files traced as server runtime dependencies are bundled into `.next/standalone/`).
- `public/` exists in the repo (verified — contains `fonts/`, `hero.jpg`, `logo.svg`, `og-image.png`, `apple-touch-icon.png`, `favicon.ico`, `icon.png`, `maps/`). No conditional needed.
- `&&` short-circuits: if either copy fails, `node server.js` never runs and Playwright's webServer wait will surface the real error in the GitHub Actions log rather than masking it as a timeout.
- `exec` not needed — npm/pnpm scripts already spawn under `sh -c` and pass the final process up; SIGTERM from Playwright's webServer teardown reaches `node server.js`.

Do NOT change `"start": "next start"` — it stays as the local-dev convenience alias (and matches what `payload` admin docs assume). The standalone variant is opt-in.

**Step 2 — Edit `playwright.config.ts` lines 38-43.**

Change the `webServer.command` so the CI branch invokes the new script:

```ts
// Always boot a local server. In CI, `pnpm build` has already produced `.next/standalone/`
// so we boot the standalone server entry directly (Next.js refuses `next start` under
// `output: 'standalone'`); locally `pnpm dev` keeps HMR. All required env vars
// (`DATABASE_URL`, `AUTH_SECRET`, `OTP_HMAC_KEY`, Turnstile keys, etc.) are inherited
// from the GitHub Actions job-level `env:` block.
webServer: {
  command: process.env.CI ? 'pnpm start:standalone' : 'pnpm dev',
  url: 'http://localhost:3000',
  reuseExistingServer: !process.env.CI,
  timeout: 120_000,
},
```

Only the `command` line and the explanatory comment above it change. `url`, `reuseExistingServer`, and `timeout` are unchanged. Leave all `projects[]`, `use`, `reporter`, etc. untouched.

**Out of scope (do NOT touch):**
- `next.config.ts` — keep `output: 'standalone'`.
- `.github/workflows/ci.yml` — no build-step changes, no env changes.
- Any test file under `tests/`.
- The `start` script — leave as `next start`.
