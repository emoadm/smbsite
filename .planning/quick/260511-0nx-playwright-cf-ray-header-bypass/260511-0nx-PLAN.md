From src/middleware.ts:38-52 (the cf-ray casual-probe gate):
```typescript
export function middleware(req: NextRequest) {
  // Allow all traffic in non-production environments (dev, test, preview deploys).
  if (process.env.NODE_ENV !== 'production') {
    return NextResponse.next();
  }
  // Cloudflare always adds cf-ray to proxied requests. Direct origin-IP
  // access (bypassing Cloudflare) won't have it — block as a casual-probe gate.
  if (!req.headers.get('cf-ray')) {
    return new NextResponse('Forbidden', { status: 403 });
  }
  return NextResponse.next();
}
```

Key point: the gate is a **presence check only** (line 48). Format is NOT validated. Any non-empty string passes. Using a clearly synthetic value (`playwright-ci-bypass`) instead of a forged-looking real cf-ray (`8a1b2c3d-LHR`) keeps log inspection honest — it's obvious this is CI test traffic, not a forged Cloudflare request.

Also note `process.env.NODE_ENV !== 'production'` early-return: locally where `pnpm dev` sets NODE_ENV=development, the header would not be required. The header IS required in CI because `pnpm start:standalone` (started by Playwright's webServer in CI) runs `node .next/standalone/server.js` against a `pnpm build` output where NODE_ENV=production.
