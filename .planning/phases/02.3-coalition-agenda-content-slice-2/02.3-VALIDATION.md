## Test Infrastructure

| Property | Value |
|----------|-------|
| **Unit framework** | Vitest 3.x (`vitest run`) — config `vitest.config.mts` |
| **E2E framework** | Playwright (`playwright test`) — config `playwright.config.ts` (4 viewport projects: desktop 1440×900, tablet 768×1024, iPhone SE 375×667, Pixel 5 360×740) |
| **Quick run command** | `pnpm vitest run tests/unit/agenda-toc-anchors.test.ts && pnpm typecheck && pnpm lint:i18n` |
| **Full suite command** | `pnpm test:unit && pnpm test:e2e && pnpm lint:i18n && pnpm lint:tokens && pnpm check:placeholders && pnpm typecheck` |
| **Estimated quick runtime** | ~12–18 seconds |
| **Estimated full runtime** | ~90–120 seconds (Playwright dominates) |

> **CRITICAL invocation note** (from Phase 02.2 SUMMARY D-4): `pnpm test --run X` FAILS — pnpm consumes `--run`. Use `pnpm vitest run X` directly OR `pnpm test:unit X` (since `test:unit` script = `vitest run`, extra args after the script name pass through correctly).

---

## Sampling Rate

- **After every task commit:** Run quick command above (~15s feedback latency).
- **After every slice merge (3 slices total):** Above + `pnpm check:placeholders` + `pnpm test:e2e tests/e2e/responsive.spec.ts` for any `/agenda`-touching changes.
- **Before `/gsd-verify-work`:** Full suite must be green (all four lint scripts + both test runners + typecheck).
- **Operator visual gate (manual, AGENDA-S2-05):** 14-step walkthrough on desktop + mobile viewports (template inherited from Phase 02.2 SUMMARY).
- **Max feedback latency:** 18 seconds (quick command).
