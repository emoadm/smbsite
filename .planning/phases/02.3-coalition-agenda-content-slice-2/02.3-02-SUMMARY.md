1. **Образование sub-section `<h3>` text length:** Both sub-section headings are multi-sentence descriptions in the source (not short labels). Rendered verbatim as multi-line `<h3>` text per UI-SPEC §3.3.F source-fidelity rule. Neither has an `id=` attribute per §3.3.A.

2. **Социална политика list counts:** Planner estimated 4/5/4 items in sub-sections 1, 3, 5. Source-verified: Връщане на личната отговорност = 4 items (lines 680-691), Политика на пазара на труда = 5 items (lines 710-720), Пенсионна система = 4 items (lines 739-742). Estimates were correct.

3. **Култура item 1 nested `<ul>`:** The plan stated "items 6-10 each contain a nested `<ul>`" but the source (lines 1152-1161) shows item 1 ("Българската културна идентичност") also has 3 em-dash sub-bullets under "Стъпки за постигане на тази цел:". Rendered as nested `<ul>` per source-fidelity rule — deviation from plan estimate, but correct per source. Items 2-5 verified to have plain prose only (no nested sub-bullets).

4. **Култура em-dash bullet stripping:** All `–` characters were stripped from `<li>` text in nested `<ul>` blocks. The `grep -nE '<li>\s*–'` check returns 0 matches, confirming no em-dash text remains as `<li>` content.
