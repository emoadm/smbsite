## Shared Patterns

### Source-fidelity rule (applies to ALL chapter authoring)

**Source:** UI-SPEC §3.3.F + L-18; live `src/app/(frontend)/agenda/page.tsx` slice-1 ship as the precedent
**Apply to:** every new `<section>` in `page.tsx`

```text
1. Em-dashes (–), curly quotes („…"), NBSPs preserved verbatim.
2. pdftotext line-de-wrap rule:
   • blank line in source = paragraph break = new <p>
   • consecutive non-blank lines = same paragraph = joined into one <p>
3. Source's restarted "1." numbering (pdftotext page-break artifact) is corrected
   via real <ol> semantics — re-numbered 1-N in rendered HTML.
4. Source's em-dash bullet markers (–) ARE stripped from rendered text when they
   are bullet glyphs (Култура items 6-10 nested <ul>). The glyph is NOT prose.
5. NEVER reword, summarize, or rewrite coalition prose. Verbatim or nothing.
```

### `prose` reliance — no spacing or typography overrides

**Source:** UI-SPEC §3.2 + L-10
**Apply to:** every new chapter `<section>` and every nested element inside `<article>`

```text
ALLOWED overrides on JSX inside the <article>:
  • id="…" on <h2> elements (anchor targets — locked IDs only)
  • <section> wrapping for semantic grouping (no styling effect)

FORBIDDEN overrides:
  • mt-*, mb-*, py-*, px-* on <p>, <h2>, <h3>, <ul>, <ol>, <li>, <strong>
  • text-*, font-*, leading-*, tracking-* on any prose element
  • list-disc, list-inside, list-decimal — prose supplies markers
  • className="lead" or any "blockquote-like" wrapper for source's left-indented
    lead paragraphs (UI-SPEC §3.3.E lock)
```
