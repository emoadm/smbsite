- **`payload migrate` is BLOCKED** (project MEMORY: project_payload_schema_constraint.md, feedback_gsd_141_sdk_patch.md). All DDL applied as **manual SQL via Neon SQL Editor**. The Drizzle migration file is generated by `pnpm drizzle-kit generate` and committed; the apply step is a `[BLOCKING] autonomous: false` task in this plan.
- **No `pgEnum`.** All enum-shaped columns are `text` with Zod enforcement at the API boundary. Verified anti-pattern in `auth.ts:3` and `attribution.ts:21` comments.
- **Two separate role concepts.** `users.role` keeps business-role values (D-10: 'owner'|'manager'|'employee'|'other') — do NOT touch. The new `users.platform_role` column (NULLable, default NULL) holds 'editor'|'super_editor' for editorial users on the application identity. Payload `admin_users.role` is a SEPARATE table (`src/collections/Users.ts`) — keep both in sync via the ops runbook (delivered in Plan 04-07). PATTERNS.md "Critical Architecture Finding: Two Separate User Tables" is the authoritative explanation.
- **moderation_log is Drizzle-managed, NOT a Payload collection.** Payload UI on a moderation_log collection would expose Edit buttons and prevent DB-level REVOKE protection (Pitfall 1).
- **No raw `submitter_id`/`full_name`/`email` on any public surface.** D-C1 absolute lock — public schema exposure path is constrained at the query layer in Plan 04-05.
