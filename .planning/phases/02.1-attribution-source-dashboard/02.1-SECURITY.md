# Phase 02.1 — Attribution Source Dashboard Security

> Per-phase security contract: threat register, accepted risks, and audit trail.

---

## Trust Boundaries

| Boundary | Description | Data Crossing |
|----------|-------------|---------------|
| application code → Postgres | Drizzle ORM serializes column declarations to DDL; a misnamed column (e.g. `inet`) would persist raw IPs | Schema source |
| schema source file → developer | Future plan or refactor could add raw-IP columns | Column declarations |
| Docker build → MaxMind download.maxmind.com | License key + mmdb file cross HTTPS at build time only; runtime IPs never leave the EU container | Build secret |
| `MAXMIND_LICENSE_KEY` build-secret → image layers | Mounted to `/run/secrets/...` and discarded after RUN; not baked into image layers | Secret key |
| `GEOIP_DB_PATH` env var → Reader.open() | A maliciously-set path could point Reader at an attacker-controlled mmdb file | File path |
| messages/bg.json → JSX render | All public copy; renderers call `t(key)`; hardcoded JSX strings bypass i18n + grep gates | Display strings |
| Job payload (Redis) → worker (Node process) | `raw_ip` lives in Redis as JSON for at most `removeOnComplete: 24h` | Ephemeral PII |
| Worker → Pino logs → Logtail/Better Stack EU aggregator | Pino REDACT prevents accidental raw_ip / IP leakage | PII (blocked) |
| Visitor browser → /api/attr/init | First-party fetch carrying URL params + Cookie + UA + cf-connecting-ip | Anonymous attribution data |
| /api/attr/init → addAttributionJob → Upstash Redis | Node-runtime BullMQ producer pushes job; worker consumes asynchronously | Job payload |
| Cloudflare cache → public landing pages | Public pages MUST NOT have Set-Cookie (D-22 invariant) | Session cookie |
| Visitor browser → register Server Action | Form data crosses with attr_sid cookie automatically attached | Registration PII |
| visitor browser → verifyOtp Server Action | attr_sid cookie crosses; verifyOtp UPDATEs attribution_events on success | Anonymous-to-user link |
| Payload admin browser → /admin/views/attribution RSC | Payload session cookie carries identity; req.user populated by initPageResult | Aggregate data |
| Server Action call (client → fetchAttributionAggregates) | RSC + Server Actions share auth via cookies; assertEditorOrAdmin re-validates | Aggregate query |
| Hetzner runner → staging.chastnik.eu | Ephemeral load-test runner; teardown after test | Load test traffic |
| Staging OTP fixture → staging DB | Intentionally weak credential scoped to staging; cleanup mandated post-run | Test credentials |
