## Pattern Assignments

### `src/db/schema/attribution.ts` (model, CRUD)

**Analog:** `src/db/schema/consents.ts` (lines 1–31)

**Imports pattern** (`consents.ts` lines 1–2):
```typescript
import { pgTable, uuid, text, boolean, timestamp, index } from 'drizzle-orm/pg-core';
import { users } from './auth';
```
For `attribution.ts` drop `boolean`, keep `uuid, text, timestamp, index`:
```typescript
import { pgTable, uuid, text, timestamp, index } from 'drizzle-orm/pg-core';
import { users } from './auth';
```

**FK + cascade pattern** (`consents.ts` lines 18–21):
```typescript
user_id: uuid('user_id')
  .notNull()
  .references(() => users.id, { onDelete: 'restrict' }),
```
Attribution uses `onDelete: 'cascade'` (not `restrict`) per D-07 GDPR-05 forward-prep:
```typescript
user_id: uuid('user_id')
  .references(() => users.id, { onDelete: 'cascade' }),  // nullable until email confirm
```

**Index pattern** (`consents.ts` lines 27–29):
```typescript
(t) => ({
  userKindIdx: index('consents_user_kind_idx').on(t.user_id, t.kind),
}),
```
Attribution needs five indexes (sid, user, first_seen, utm_source, oblast) — copy the `(t) => ({})` shape and add all five.

**Column type discipline:** All attribution columns use `text()` — NOT `pgEnum`. Matches the existing `sector: text('sector').notNull()` pattern in `src/db/schema/auth.ts` line 16. The rule: pg enums require DDL to add values; text + Zod-enforced enum at the API boundary is the project convention (see auth.ts comment lines 3–4).

**Critical constraint — NO inet column:** No `inet`, no `text('raw_ip')`, no `text('ip')`. Unit test in `tests/unit/attribution-schema.test.ts` must grep `src/db/schema/attribution.ts` and assert neither string appears.

**Queue singleton pattern** (`email/queue.ts` lines 29–33):

Replace `EmailJobPayload` with `AttributionJobPayload` and `EMAIL_QUEUE_NAME` with `ATTRIBUTION_QUEUE_NAME`.

**Job add pattern with test bypass** (`email/queue.ts` lines 36–78):
```typescript
export async function addEmailJob(payload: EmailJobPayload): Promise<void> {
  // TEST_OTP_SINK dev shortcut (omit for attribution — no file sink needed)
  const url = process.env.UPSTASH_REDIS_URL;
  const isTestBuild = process.env.NEXT_PUBLIC_TURNSTILE_SITE_KEY === '1x00000000000000000000AA';
  const isLocalUrl = !!url && (url.includes('localhost') || url.includes('127.0.0.1'));
  if (!url || isLocalUrl || isTestBuild) {
    if (process.env.NODE_ENV === 'production' && !isTestBuild && !isLocalUrl) {
      throw new Error('UPSTASH_REDIS_URL must be set in production');
    }
    return;
  }
  await getQueue().add(payload.kind, payload, {
    attempts: 5,
    backoff: { type: 'exponential', delay: 5_000 },
    removeOnComplete: { age: 24 * 3600, count: 1000 },
    removeOnFail: { age: 7 * 24 * 3600 },
  });
}
```
For attribution: use `attempts: 3` (Claude's discretion — attribution failure is never user-visible, so fewer retries than email). Keep `backoff`, `removeOnComplete`, `removeOnFail` identical.

**Job name:** Use `payload.attr_sid` as the job name (analogous to `payload.kind` for email). Or use a fixed string `'attribution-event'`. Either works; job name is for BullMQ UI display only.

**closeQueue pattern** (`email/queue.ts` lines 81–86): copy verbatim, renamed to `closeAttributionQueue`.

**Type definitions:** Export `AttributionJobPayload` interface at top of file (mirrors `EmailJobPayload` lines 4–12). Fields per D-08: `attr_sid`, `raw_ip`, `ua`, `utm_source`, `utm_medium`, `utm_campaign`, `utm_term`, `utm_content`, `referer`, `landing_path`, `ts`.

async function processor(job: Job<AttributionJobPayload>): Promise<void> {
  const { attr_sid, raw_ip, utm_source, utm_medium, utm_campaign,
          utm_term, utm_content, referer, landing_path, ts } = job.data;
  // 1. GeoIP lookup (raw_ip discarded after this step)

Add `raw_ip` to the redact list in `src/lib/logger.ts` alongside `ip` (lines 3–11). Alternatively, never pass `raw_ip` to logger calls — preferred because it doesn't widen the redact list for unrelated modules.

**startWorker export pattern** (`email/worker.tsx` lines 99–107):
