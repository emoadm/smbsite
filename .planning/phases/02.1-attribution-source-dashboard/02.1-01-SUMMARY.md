- **Migration approach: db:generate not db:push (operator-deferred live apply).** The worktree executor has no `DIRECT_URL` env var (verified via `node -e`), so the plan's primary path (`pnpm db:push` straight to Neon staging) is unreachable. The grep test strips `^//` lines before checking so the documentation cannot trip the test. The plan's <action> block called this out explicitly and the test mirrors that note.

I followed the explicit column list (which is unambiguous and matches what every Wave 2 plan will INSERT/UPDATE). Task 3's verify regex `^(2[3-9]|[3-9][0-9])$` accepts 25, so no gate failure.
