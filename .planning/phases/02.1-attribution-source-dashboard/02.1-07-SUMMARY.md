# Phase 02.1 Plan 07: Attribution Dashboard (Payload Admin View) Summary

Editor-facing attribution dashboard built as a Payload custom admin view at `/admin/views/attribution`. Role-gated to admin + editor; 4 Drizzle aggregate queries (by UTM source, oblast, QR flag, self-reported source via INNER JOIN to users); bar list for sessions-by-oblast using Bulgarian display names from `OBLAST_NAMES`; date-range filter (default last 7 days); CSV export with RFC 4180 quoting.

## Aggregate Query List

| # | Output | Source | GroupBy |
|---|--------|--------|---------|
| 1 | byUtmSource | attribution_events | first_utm_source |
| 2 | byOblast | attribution_events | first_oblast |

All 5 share the same `buildWhere(filter)` predicate — date range + optional oblast/utmSource/qrFlag filters.

## CSV Export

- Header row: `attr_sid,first_utm_source,first_utm_medium,first_oblast,first_qr_flag,first_seen_at,user_id`
- Quoting: values containing `"`, `,`, or newline get wrapped in `"..."`; embedded quotes doubled.
- Filename: `attribution-{dateFromYYYY-MM-DD}-{dateToYYYY-MM-DD}.csv`
- Order: `ORDER BY first_seen_at` ASC.

## next-intl Outcome

**Direct JSON import used.** Confirmed Payload admin shell does NOT provide a next-intl Context — the existing pattern in `src/lib/email/worker.tsx` (worker context, also outside provider) uses the same approach. Both `page.tsx` and `AttributionDashboard.tsx` import `messages/bg.json` and read the `attribution.dashboard.*` namespace as a plain object. Trade-off: TypeScript can't enforce key existence (we use type assertions for safety), but runtime is fast and there's no provider plumbing.

## Role Field Confirmation

**Singular `role` confirmed correct** for this codebase. Source of truth: `src/collections/Users.ts` lines 22-30 — `admin_users.role` is a `select` field with options `[admin, editor]`. PATTERNS.md / RESEARCH.md mention of `roles` (plural) is incorrect for this project. Tests enforce this with a negative grep:


## Deviations from Plan

1. **@payloadcms/ui dep added** — plan assumed it was importable transitively; pnpm hoisting required promoting to direct dep. Single-line `pnpm add @payloadcms/ui@3.84.1` in main repo cwd. Documented and verified via `ls node_modules/@payloadcms/ui`.
2. **Test file moved tests/integration/ → tests/unit/** — `vitest.config.mts` only includes `tests/unit/**`. Tests are source-grep only (no DB), so unit/ is the correct destination. Plan path was incorrect; behavior unchanged.
3. **Drizzle parameterization test rewrote regex** — original `\.where\(.*eq\(` failed because `.where(where)` uses precomputed predicate; rewrote to assert `eq()/gte()/lte()` calls in `buildWhere()` instead. Same security guarantee; cleaner assertion.
