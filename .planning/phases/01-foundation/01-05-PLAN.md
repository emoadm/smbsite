## Objective

Wire Auth.js v5 (beta.31) with the `@auth/drizzle-adapter`, custom 6-digit OTP generation, **HMAC-SHA256 token hashing** (Pitfall K — security hardening required for low-entropy OTPs), database-backed sessions with the 30-day sliding cookie (D-03), the email-verification middleware gate (AUTH-04), and the App Router auth handler. The `sendVerificationRequest` callback enqueues to BullMQ in plan 1.10 — for now it stages the integration with a `TODO(plan-10)` placeholder that throws if invoked, ensuring no email is sent before the queue is wired.

## Truths

- D-01: Login = 6-digit numeric OTP. No magic links.
- D-02: `session.strategy: 'database'` via `@auth/drizzle-adapter`. NO JWT.
- D-03: `maxAge: 30*24*60*60` (30 days). `updateAge: 24*60*60` (sliding refresh on each visit).

- D-07: OTP-verify rate limit = 5 attempts per OTP before invalidation. Implemented via the `attempts` column on `verification_tokens` from plan 1.03.
- RESEARCH § Auth.js v5 Email OTP Configuration (lines 461-521): override `generateVerificationToken` for 6-digit code; route `sendVerificationRequest` through queue.

- RESEARCH Pitfall K (lines 1326-1331): Auth.js adapter stores tokens plaintext by default. We override storage to keep HMAC-SHA256 hashes.
- RESEARCH § Email Verification Gate (lines 538-561): middleware redirects `/member/*` to `/login` if `!emailVerified`.

    ```typescript
    import crypto from 'node:crypto';

    /** Generate a 6-digit numeric OTP using crypto-secure randomness (D-01, AUTH-03). */
    export function generateOtpCode(): string {
      // crypto.randomInt is uniform; range [0, 1_000_000)
      const n = crypto.randomInt(0, 1_000_000);
      return String(n).padStart(6, '0');
    }

    /** HMAC-SHA256(identifier:code, OTP_HMAC_KEY) → 64 hex chars. (Pitfall K) */
    export function hashOtp(code: string, identifier: string): string {
      const key = process.env.OTP_HMAC_KEY;
      if (!key) throw new Error('OTP_HMAC_KEY not configured');
      return crypto.createHmac('sha256', key).update(`${identifier}:${code}`).digest('hex');
    }

    /** Constant-time comparison of submitted code vs stored hash. */
    export function verifyOtpHash(submittedCode: string, identifier: string, storedHash: string): boolean {
      const computed = hashOtp(submittedCode, identifier);
      const a = Buffer.from(computed, 'hex');
      const b = Buffer.from(storedHash, 'hex');
      if (a.length !== b.length) return false;
      return crypto.timingSafeEqual(a, b);
    }

    /** D-04: registration OTPs are valid 48 hours. */
    export function registrationOtpExpiry(): Date {
      return new Date(Date.now() + 48 * 60 * 60 * 1000);
    }

    /** D-04: login OTPs are valid 10 minutes. */
    export function loginOtpExpiry(): Date {
      return new Date(Date.now() + 10 * 60 * 1000);
    }

    /** D-07: a single OTP can be tried at most 5 times. */
    export const MAX_OTP_VERIFY_ATTEMPTS = 5;
    ```

    ```typescript
    export async function persistHashedOtp(
      identifier: string,
      plaintextCode: string,
      kind: 'register' | 'login',
    ): Promise<{ expires: Date }> {
      const expires = kind === 'register' ? registrationOtpExpiry() : loginOtpExpiry();

          token: tokenHash,
          expires,
          kind,
          attempts: '0',
        });
      });

      return { expires };
    }
    ```
