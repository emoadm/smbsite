# Phase 1 — UI Design Contract

> Visual and interaction contract for the Foundation phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Phase 1 ships only the auth and legal surface** (registration, OTP login, error pages, privacy/terms drafts, shared layout shell). The full agitation-page design system lands in Phase 2. This contract intentionally limits component depth to D-26's locked set plus the four primitives demanded by AUTH-03, D-08–D-12 (input-otp, checkbox, select, form). All copy is Bulgarian via `next-intl` from day one (D-27); no hardcoded Cyrillic in components.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | shadcn (CLI-managed, copy-paste, owned) |
| Preset | `new-york` style + `slate` base color, Tailwind v4 with CSS variables, `--prefix=""`. To be initialized via `npx shadcn@latest init` in plan task 1.x. |
| Component library | Radix UI (primitives shipped under shadcn/ui) |
| Icon library | `lucide-react` (shadcn default; full coverage, tree-shakeable, no bespoke SVGs in Phase 1) |
| Font | **Roboto** (cyrillic + cyrillic-ext subsets) for body + UI; **Roboto Slab** (cyrillic + cyrillic-ext subsets) for display headings. Loaded via `next/font/google` with `display: swap` and explicit `subsets: ['cyrillic', 'cyrillic-ext', 'latin']`. Fallback stack: `system-ui, -apple-system, "Segoe UI", Tahoma, sans-serif`. Rationale: matches sinyabulgaria.bg's official typography and is the only cyrillic-tested pairing the coalition already uses; satisfies BRAND-06 (full Cyrillic incl. Ж, Щ, Ъ, Ю, Я, ѝ, diacritics). |

**Components in scope for Phase 1** (locked set — planner installs exactly these, no more):

| shadcn component | Rationale (requirement / decision) |
|---|---|
| `button` | All CTAs across registration, OTP, logout, error retry (D-26). |
| `input` | Email, name fields on registration / login (AUTH-01, AUTH-05, D-08). |
| `label` | Accessible field labels for every input (BRAND-06 a11y baseline, WCAG forward-prep). |
| `alert` | Error states, rate-limit notices, "проект" draft-page banner, OTP-sent confirmation, expired-OTP message (D-15, AUTH-08, AUTH-09). |
| `card` | Form containers on /register, /login, /auth/otp; legal-page wrappers (D-26 layout shell). |
| `input-otp` | AUTH-03 — 6-digit code entry; 6 slots, `inputmode="numeric"`, `pattern="\d{6}"`, auto-focus, auto-submit on slot 6, mobile-keyboard numeric. Specific UX preference from CONTEXT.md `<specifics>`. |
| `checkbox` | D-12 — 4 separate consent checkboxes, none pre-ticked. |
| `select` | D-08/D-09/D-10 — sector (5 values) and role (4 values) dropdowns. Native `<select>` is acceptable on small mobile viewports; planner picks shadcn `select` for visual consistency unless mobile UX testing reveals friction. |
| `form` | RHF + zod resolver wrapper (CLAUDE.md stack: React Hook Form 7 + zod + zod-i18n-map for Bulgarian error messages). |

No other components install in Phase 1. Phase 2 will add navigation, hero, video, accordion, etc. as the public agitation surface ships.

## Spacing Scale

Declared values (multiples of 4 only — Tailwind v4 default, no custom override needed):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gap inside buttons; checkbox-to-label gap. |
| sm | 8px | Stacked form-row gap between label and input; alert padding-y. |
| md | 16px | Default inter-field spacing; card padding on mobile; paragraph margin-bottom. |
| lg | 24px | Card padding on desktop; section padding inside `<main>`; gap between consent-checkbox group rows. |
| xl | 32px | Vertical gap between hero block and form card on /register. |
| 2xl | 48px | Header bottom margin → form; footer top padding. |
| 3xl | 64px | Page top padding on desktop; vertical rhythm between major regions on legal pages. |

**Touch targets:** all interactive elements (buttons, checkboxes, links inside footer) maintain ≥ 44×44px hit area on mobile per Apple HIG / WCAG 2.5.5 forward-prep — Phase 1 baseline, formal audit in Phase 6 (BRAND-04).

**Exceptions:** OTP input slots are visually 48×56px each (slightly taller than the 44px floor) for thumb-friendly numeric entry; total slot row width fits within a 360px viewport with 8px slot gaps + 16px separator — verified against the iPhone SE viewport from D-28's test matrix.

## Typography

Exactly **4 sizes**, **2 weights** — locked for Phase 1.

| Role | Size | Weight | Line Height | Family | Usage |
|------|------|--------|-------------|--------|-------|
| Label | 14px | 600 (semibold) | 1.4 | Roboto | Form-field labels, footer legal links, secondary metadata (e.g. "проект, последна редакция …" marker per D-15). |
| Body | 16px | 400 (regular) | 1.5 | Roboto | All paragraph text on legal pages, form helper text, alert messages, button labels (buttons use 16px regular). |
| Heading | 20px | 600 (semibold) | 1.3 | Roboto | Card titles ("Регистрация", "Влез в профила си", "Въведи кода"), section headings on legal pages (`<h2>`). |
| Display | 28px | 600 (semibold) | 1.2 | Roboto Slab | Page titles `<h1>` on /register, /login, /privacy, /terms, error/maintenance pages. Roboto Slab provides the editorial weight matching the coalition's print/letter aesthetic — visually distinct from Roboto body without introducing a third weight. |

**Rules:**
- No font-size other than the four declared above appears anywhere in Phase 1 UI. Tailwind utilities limited to `text-sm` (14), `text-base` (16), `text-xl` (20), `text-3xl` (28) — `tailwind.config` aliases or Tailwind v4 `@theme` block redefines defaults to these exact values to prevent drift.
- Numerals in OTP slots use `font-variant-numeric: tabular-nums` and `font-weight: 600` at `28px` (Display size) for unambiguous digit reading.
- Headings ALL use sentence case, never ALL CAPS. (Coalition source site uses ALL CAPS in its hero — explicitly NOT imitated per BRAND-03 "fresh modern design, not pixel-imitation".)
- Body line-height 1.5 ensures Bulgarian diacritics (ѝ, й, ъ̀) render with full vertical breathing room.

## Color

Extracted from the official Синя България logo SVG (`/media/2023/08/logo-vector-1.svg`) and Elementor kit-11 global tokens at sinyabulgaria.bg, then refit into a 60/30/10 contemporary system. Dark mode is **out of scope for Phase 1** — light only (confirmed in absence of any CONTEXT.md decision to the contrary; planner may add a `prefers-color-scheme: dark` token map in Phase 6 alongside the WCAG audit, BRAND-04).

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#FFFFFF` (white) | Page background, card surfaces, input field background. |
| Secondary (30%) | `#F1F5F9` (slate-100 — pairs with shadcn `slate` base) | Footer background, alert default surface, legal-page outer container, hover state on secondary buttons. |
| Accent (10%) | `#004A79` (deep navy from coalition logo) | Primary CTA button background, focused-input ring, active-state borders, link color (replacing sinyabulgaria.bg's lighter `#1FA4C1` — navy carries higher contrast against white surfaces, easier WCAG AA pass). |
| Destructive | `#E72E4D` (red from coalition logo) | Destructive button background; field-validation error icon + border; rate-limit-exceeded alert border-left. |

**Supporting tokens** (not part of the 60/30/10 narrative — used in narrowly-defined states only, declared here so checker can verify they don't bleed into general decoration):

| Role | Value | Usage |
|------|-------|-------|
| Success | `#009F54` (green from coalition logo) | "Кодът е изпратен" success alert; "Регистрацията е потвърдена" confirmation page accent; `aria-invalid="false"` field check icon. |
| Sky (link hover / secondary highlight) | `#00B7ED` (sky-blue from coalition logo) | Hover state on accent links only (never as a fill). |
| Foreground (text on white) | `#0F172A` (slate-900) | All body and heading text on white surfaces. WCAG AA contrast against `#FFFFFF` = 16.8:1. |
| Muted foreground | `#475569` (slate-600) | Helper text, draft-page "проект" marker, footer text. WCAG AA against `#FFFFFF` = 7.2:1. |
| Border (default) | `#E2E8F0` (slate-200) | Card borders, input borders (default state), divider lines. |
| Border (focus) | `#004A79` (Accent) | Input focus ring, button focus ring (3px outer ring, 2px offset per shadcn default). |

**Accent reserved for** (explicit list — accent never appears on general decoration):
1. Primary CTA buttons: "Регистрирай се", "Изпрати код", "Влез", "Потвърди", "Запази".
2. Active page indicator in header navigation (when added — Phase 1 header has no nav links beyond logo + login/logout, so no active state yet).
3. Link color in body text and footer.
4. Focus ring on all interactive elements (input, button, checkbox, OTP slot).
5. Active-step indicator in any multi-step flow (registration → OTP confirmation arrow icon).
6. Logo monogram color in header when the SVG is rendered as a single-color CSS mask (only valid alternate; full-color SVG logo is the default).

**Destructive reserved for** (explicit list — never used decoratively):
1. The "Изтрий" / "Прекъсни регистрацията" button (Phase 1 has no destructive flow yet — listed for Phase 6).
2. Field-level validation error border + error message text.
3. Rate-limit-exceeded alert border-left and icon.
4. Disposable-email-rejected alert (per D-06).

**WCAG AA contrast verification** (Phase 1 baseline; formal audit Phase 6):
- Body text `#0F172A` on `#FFFFFF`: 16.8:1 (AAA). ✓
- Body text `#0F172A` on `#F1F5F9`: 15.6:1 (AAA). ✓
- Accent button text `#FFFFFF` on `#004A79`: 9.8:1 (AAA). ✓
- Link text `#004A79` on `#FFFFFF`: 9.8:1 (AAA). ✓
- Destructive text `#FFFFFF` on `#E72E4D`: 4.7:1 (AA large text — borderline for body; planner uses `#C71F3D` for destructive button text background, 5.9:1, if visual QA flags. Spec primary remains `#E72E4D` from logo extraction; alternate noted.).
- Muted foreground `#475569` on `#FFFFFF`: 7.2:1 (AAA). ✓
