## Objective

Stand up Drizzle ORM against Neon Postgres (Frankfurt) with TWO connection strings (pooled `DATABASE_URL` for app queries, direct `DIRECT_URL` for migrations per RESEARCH Pitfall C). Define the four Auth.js v5 adapter tables with project-extension columns from D-08/D-11, plus the append-only `consents` table from D-13. Generate the first SQL migration and `[BLOCKING]` push it to the Neon database so all downstream Wave-1 work runs against a real schema.

## Truths

- D-02: Auth.js v5 + database sessions via `@auth/drizzle-adapter`. No JWT.
- D-08: Registration fields = `full_name`, `email`, `sector`, `role`, plus consent rows.
- D-09: Sector enum values — `ИТ`, `Търговия`, `Производство`, `Услуги`, `Друго`.
- D-10: Role enum values — `Собственик`, `Управител`, `Служител`, `Друго`.
- D-11: Phase 1 stores ONLY `full_name`. No `display_name` column. Phase 3 adds it.
- D-13: `consents` is append-only — `(id, user_id, kind, granted, version, granted_at, region)` with `kind` enum covering `privacy_terms | cookies | newsletter | political_opinion`. `onDelete: 'restrict'` on the FK to `users.id` (Phase 6 changes the deletion semantics).
- D-04: `verification_tokens` carries `kind` column (`register` 48h | `login` 10min) — Phase 1 stores OTP tokens HASHED (Pitfall K, plan 05).
- RESEARCH § Pitfall C: `drizzle-kit migrate/push` must use `DIRECT_URL`, never the pooled URL.
- RESEARCH § Neon Postgres Connection Pooling: use `neon-serverless` (WebSocket Pool) for the main `db` instance — Auth.js sessions need transactions.

    **Step 4.** Create `src/db/schema/consents.ts` (D-13, append-only):
    ```typescript
    import { pgTable, uuid, text, boolean, timestamp, index } from 'drizzle-orm/pg-core';
    import { users } from './auth';

    export const CONSENT_KINDS = [
      'privacy_terms',
      'cookies',
      'newsletter',
      'political_opinion',
    ] as const;
    export type ConsentKind = (typeof CONSENT_KINDS)[number];

    // D-13: append-only. NEVER UPDATE or DELETE rows from the application.
    // Withdrawals INSERT a new row with granted = false.
    // onDelete: 'restrict' — Phase 6 deletion flow handles cascade explicitly.
    export const consents = pgTable(
      'consents',
      {
        id: uuid('id').defaultRandom().primaryKey(),
        user_id: uuid('user_id').notNull().references(() => users.id, { onDelete: 'restrict' }),
        kind: text('kind').notNull(),    // one of CONSENT_KINDS
        granted: boolean('granted').notNull(),
        version: text('version').notNull(),    // policy version, e.g. "2026-04-29"
        granted_at: timestamp('granted_at', { withTimezone: true }).defaultNow().notNull(),
        region: text('region'),    // populated in Phase 2 (oblast/country)
      },
      (t) => ({
        userKindIdx: index('consents_user_kind_idx').on(t.user_id, t.kind),
      }),
    );
    ```
