## Objective

Bootstrap the project from a clean working directory into a runnable Next.js 15.3.x + Payload CMS 3.84.x + TypeScript scaffold with strict ESLint/Prettier, the `@/*` path alias, the canonical directory layout from `01-PATTERNS.md`, and a complete `.env.example`. This plan is the foundation everything else depends on. **No business logic ships here** — only the runnable shell.

## Truths

- D-25: Payload first-user-creates-admin gated to localhost-only on first deploy. No public admin signup.
- RESEARCH Q1 resolution: pin **Next.js 15.3.x** (not 16.x) for Phase 1 stability. Payload 3.84.1 + Auth.js v5 beta + Drizzle 0.45 are all verified against this combination.
- RESEARCH Pitfall A: env-var prefix is `AUTH_*` not `NEXTAUTH_*`. Pin `next-auth@5.0.0-beta.31` exactly (no caret).
- RESEARCH Pitfall B: Payload 3.84 + Next 15 + Turbopack is unstable. Use **webpack in dev** (do not pass `--turbopack`).
- PATTERNS § Directory Layout: `src/app/(frontend)`, `src/app/(payload)`, `src/db`, `src/lib`, `src/components`, `messages/`, `tests/` are canonical. Lock these now.
- PATTERNS § File Naming: source files kebab-case, components PascalCase, message keys `dot.notation.camelCase`.
- All commands run via **pnpm** — frozen lockfile in CI.
