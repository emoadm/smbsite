## 2. Spacing Scale

8-point scale (multiples of 4 only). Maps to Tailwind v4 default `--spacing-*` tokens — no custom redefinition needed:

| Token | Value | Tailwind | Usage |
|-------|-------|----------|-------|
| 2xs | 4px | `gap-1`, `p-1` | Icon-to-label gaps inline |
| xs | 8px | `gap-2`, `p-2` | Compact list spacing, badge padding |
| sm | 12px | `gap-3`, `p-3` | Form field internal padding (existing) |
| md | 16px | `gap-4`, `p-4` | Default block gap, mobile section pad |
| lg | 24px | `gap-6`, `p-6` | Card padding, paragraph block gap |
| xl | 32px | `gap-8`, `p-8` | Inter-section gap on cards |
| 2xl | 48px | `gap-12`, `py-12` | Major landing section breaks (mobile) |
| 3xl | 64px | `gap-16`, `py-16` | Major landing section breaks (md+) |
| 4xl | 96px | `py-24` | Hero vertical pad on desktop |

**Section vertical rhythm on landing page:** `py-12 md:py-20` between major sections (consistent across hero/problem/vision/cta/faq-teaser). Hero gets `py-16 md:py-24`.

**Exceptions:**
- Touch targets must be ≥44×44 CSS px regardless of declared size (WCAG 2.5.5). Icon-only buttons in header use `h-11 w-11` minimum on mobile.
- Cookie banner button row uses `gap-3` (12px) per CookieYes default density to stay within their CSS overrides.

## 3. Typography

### 3.1 Font loading

Roboto is currently loaded via `next/font/google` in `(frontend)/layout.tsx`. **Phase 2 changes:**

1. **Drop `Roboto_Slab`** from layout.tsx import — replaced by Gilroy ExtraBold for display.
2. **Add Gilroy via `next/font/local`** loading the self-hosted .woff2 files. Place files at `public/fonts/`:
   - `gilroy-extrabold.woff2` — weight 800
   - `gilroy-light.woff2` — weight 300 (reserved use)
3. Free-distribution canonical source: [github.com/repalash/gilroy-free-webfont](https://github.com/repalash/gilroy-free-webfont) (Light + ExtraBold, free for personal & commercial — confirmed via project README). Plan-phase **must verify license one more time** and document the source path in plan task.
4. Roboto stays at weights 400 + 600, subsets `cyrillic`, `cyrillic-ext`, `latin`. Italic NOT included (no italic in design).

**`@font-face` is not used directly** — instead `next/font/local` is used so Next.js handles preload, fallback metrics, and FOIT/FOUT. CSS variable hookup:

```css
/* globals.css @theme block */
--font-sans: var(--font-roboto), system-ui, -apple-system, "Segoe UI", Tahoma, sans-serif;
--font-display: var(--font-gilroy), var(--font-roboto), Georgia, serif;
```

The fallback chain ensures Cyrillic still renders if Gilroy fails (Gilroy free build's Cyrillic glyph coverage is **partial** — Roboto fallback covers gaps).

### 3.2 Typographic ramp

**Locked to 4 base sizes** (no proliferation). All sizes use `rem` so the scale respects user font-size preference (accessibility).

| Role | Mobile size | Desktop size | Weight | Line-height | Letter-spacing | Font |
|------|-------------|--------------|--------|-------------|----------------|------|
| Display (hero h1) | 2.25rem (36px) | 3.75rem (60px) | 800 | 1.05 | -0.02em | Gilroy ExtraBold |
| Heading 1 (page title) | 1.875rem (30px) | 2.5rem (40px) | 800 | 1.15 | -0.015em | Gilroy ExtraBold |
| Heading 2 (section) | 1.5rem (24px) | 2rem (32px) | 800 | 1.2 | -0.01em | Gilroy ExtraBold |
| Heading 3 (subsection) | 1.25rem (20px) | 1.5rem (24px) | 600 | 1.3 | 0 | Roboto 600 |
| Body lead (sub-headline / hero kicker) | 1.125rem (18px) | 1.25rem (20px) | 400 | 1.55 | 0 | Roboto 400 |
| Body | 1rem (16px) | 1rem (16px) | 400 | 1.6 | 0 | Roboto 400 |
| Body small / caption | 0.875rem (14px) | 0.875rem (14px) | 400 | 1.5 | 0 | Roboto 400 |
| Label / form field | 0.875rem (14px) | 0.875rem (14px) | 600 | 1.4 | 0 | Roboto 600 |
| Footer microcopy | 0.8125rem (13px) | 0.8125rem (13px) | 400 | 1.5 | 0 | Roboto 400 |

**Display-only (Gilroy Light 300):** reserved for ≥4rem (64px+) numerals or single-word callouts. **Not used in Phase 2** (no such element exists in the locked landing structure). Token defined for forward-compat only.

### 3.4 Cyrillic & encoding

- Both Roboto (Google) and Gilroy (free build) MUST be tested with these glyphs before merge: **А Б В Г Д Е Ж З И Й К Л М Н О П Р С Т У Ф Х Ц Ч Ш Щ Ъ Ь Ю Я** (uppercase) + **ж щ ъ ю я** (lowercase) per BRAND-06 + Phase 1 Pitfall 10.
- If Gilroy free build is missing any Cyrillic glyph, the `--font-display` fallback chain renders the missing glyph in Roboto. This is acceptable for headlines because mixed-render is rare; only fully fallback-rendered headlines indicate font-load failure (separate problem).
- Test viewport: hero h1 at 60px, h2 at 32px, body at 16px — all visually inspected for cropped descenders (Я, Щ, Ц).

### 3.5 OTP slot rule (carried from Phase 1)

The existing rule for `[data-slot="input-otp-slot"]` (tabular-nums, weight 600) **is preserved**. It does not interact with this ramp.

### 4.4 Accent reserved-for list (the EXPLICIT 10% rule)

Accent (`--color-primary` = `#004A79`) is reserved for, and ONLY for:
1. Primary CTA buttons (`<Button variant="default">` — Регистрирай се, Присъедини се)
2. h1 and h2 text on light surfaces
3. Inline link text in body (`<a class="text-primary hover:underline">`)
4. Active nav state in header (logged-in indicator chip border)
5. Active accordion item chevron in FAQ
6. Focus ring (matched via `--color-ring`) on all interactive elements
7. Logo monochrome fallback (when SVG renders without color)

**Accent is NOT used for:** body text (uses `--color-foreground`), all interactive elements indiscriminately, decorative borders on every card, icon strokes by default, dividers.

Sky (`--color-secondary` = `#3AC7FF`) is reserved for:
1. Hero kicker line ("Платформа на МСП" type label above headline)
2. Decorative underline accent under section h2 (1 per section maximum)
3. Hover state on body links (`text-primary hover:text-secondary`) — optional; subtle
4. Outlined secondary CTA on hero ("Виж програмата" → `/agenda`)

### 4.5 WCAG AA contrast verification

All token combinations the executor will use, verified ≥4.5:1 for body, ≥3:1 for large text and UI:

| Pair | Contrast | Pass |
|------|----------|------|
| `#0F172A` foreground on `#FFFFFF` background | 19.3:1 | AA + AAA |
| `#475569` muted-foreground on `#FFFFFF` background | 8.0:1 | AA + AAA |
| `#475569` muted-foreground on `#F1F5F9` surface | 7.5:1 | AA + AAA |
| `#FFFFFF` primary-foreground on `#004A79` primary (CTA) | 9.6:1 | AA + AAA |
| `#004A79` primary on `#FFFFFF` background (link, h1) | 9.6:1 | AA + AAA |
| `#004A79` primary on `#F1F5F9` surface | 9.0:1 | AA + AAA |
| `#FFFFFF` text on `#3AC7FF` sky (button) | 1.7:1 | **FAIL** — DO NOT use white text on sky |
| `#003A60` secondary-foreground on `#3AC7FF` sky | 6.0:1 | AA |
| `#FFFFFF` on `#DC2626` destructive | 5.9:1 | AA |
| `#0F172A` on `#3AC7FF` (sky as decorative bg) | 11.4:1 | AA + AAA |

**Rule:** Sky (`#3AC7FF`) is used as **bg with dark text** OR **dark text on white**. Never `white-on-sky`.

### 4.6 Backcompat with Phase 1

Phase 1 wrote: `--color-accent: #004A79`, `--color-destructive: #E72E4D`, `--color-success: #009F54`, `--color-sky: #00B7ED`.

Phase 2 changes:

- `--color-destructive` retuned `#E72E4D` → `#DC2626` (Phase 1's `#E72E4D` is 4.5:1 — barely passes; `#DC2626` is 5.9:1 with bigger margin against future text-size changes).

## 10. Motion & Interaction

### 10.1 Allowed motion (D-08: subtle, formal-respectful)

| Surface | Motion | Duration | Easing | Trigger |
|---------|--------|----------|--------|---------|
| All sections on `/`, `/agenda` headings | Fade-in + slight rise (`translateY(8px) → 0`) | 400ms | `cubic-bezier(0.4, 0, 0.2, 1)` (ease-out) | IntersectionObserver crossing 10% viewport, ONCE per element. Use a `.fade-in-on-scroll` utility class or a tiny client component wrapper. |
| Accordion expand/collapse | Height transition + chevron rotation 180° | 200ms | ease-in-out | Click / keyboard. Radix default works. |
| Button hover | Background lighten 5% (CSS filter or token) | 150ms | ease-out | Hover only. No scale. |
| Link hover | Underline appear, color → `--color-secondary` (sky) | 100ms | ease-out | Hover. |
| Focus ring | Appears instantly (no animation) | 0 | n/a | Focus-visible |
| Cookie banner enter | Slide up + fade-in | 300ms | ease-out | Initial mount only. Provided by CookieYes. |
| Member welcome banner | NO motion. Static. | n/a | n/a | Page is informational, not celebratory. Confetti / bouncing elements are explicitly OUT (formal-respectful). |

### 10.2 Banned motion

- Autoplay video (D-03 video also obeys this — user-initiated only)
- Spinning carousels / sliders
- Parallax scrolling on hero
- Any infinite-loop animation (pulsing dots, animated borders)
- Hover effects that change element height/width (causes layout shift)
- Scale-on-hover larger than 102% (felt cartoonish)

### 10.3 Reduced motion fallback

```css
@media (prefers-reduced-motion: reduce) {
  *, *::before, *::after {
    animation-duration: 0.01ms !important;
    animation-iteration-count: 1 !important;
    transition-duration: 0.01ms !important;
    scroll-behavior: auto !important;
  }
}
```

The `.fade-in-on-scroll` utility detects `prefers-reduced-motion` and **renders elements visible immediately** without IntersectionObserver wait. Implement once in a small client wrapper.

### 10.4 Scroll behavior

- `html { scroll-behavior: smooth }` for anchor jumps. Auto-overridden by reduced-motion.
- Scroll-padding `scroll-padding-top: 5rem` so anchored sections don't hide under sticky header.

## 13. Performance & Caching (D-18)

### 13.1 Static generation strategy

- `/`, `/agenda`, `/faq` — fully static. Zero `dynamic = 'force-dynamic'` or session reads. Header session indicator stays a small client island (or async server component that gracefully falls back to "Вход" if session fetch is slow — rendered conditionally based on cookie presence at edge if Cloudflare can rewrite, otherwise plain SSR with revalidate=3600).
  - **Decision:** keep Header as-is (server component reading session). At Cloudflare CDN cache layer, `/`, `/agenda`, `/faq` are cached as **anonymous variants only** — authenticated users bypass cache (cookie-based vary). Acceptable trade-off: warmup recipients are mostly anonymous on first visit.
- `/member` — dynamic per-session (reads `auth()`); not cached. PUB-02 doesn't require it.

### 13.2 Cache headers

Set on `/`, `/agenda`, `/faq` responses:
```
Cache-Control: public, s-maxage=3600, stale-while-revalidate=86400
```

Cloudflare CDN config (out-of-scope for UI; planner addresses in plan task): Cache Rules to honor s-maxage on these three paths.

### 13.3 Image optimization

- Hero image: `next/image` with `priority`, `fill`, sized at 1920×1080 source, served via Bunny.net CDN OR `public/hero.jpg` for v1 simplicity.
- Logo: SVG inline (when coalition delivers high-res SVG), avoids HTTP roundtrip.
- All other images: `next/image` defaults.
- WebP/AVIF: Next.js handles automatically. Bunny.net image CDN handles automatically if used.

### 13.4 Font payload budget

- Roboto 400 + 600 subsets (cyrillic, cyrillic-ext, latin) = ~80KB combined woff2 (next/font self-hosts and trims to glyph subsets used).
- Gilroy ExtraBold + Light woff2 = ~80–120KB combined.
- Total font payload ≤ 250KB. Verified at plan-phase via `pnpm build` output.

### 13.5 LCP budget

Hero image is the LCP element. Target: LCP ≤ 2.5s on Slow 4G (PUB-02 requires 2s on Slow 4G — tighter than Web Vitals "good").
- Hero image preload via `<link rel="preload">` injected by `next/image priority`.
- Headline text rendered via `font-display: swap` (next/font default) — appears even before Gilroy loads.
