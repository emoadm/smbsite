## Objective

Land every Bulgarian UI string Phase 2 routes and components reference into `messages/bg.json` BEFORE the components that consume them are built. This honors the Phase 1 convention (zero hardcoded Cyrillic in JSX, all strings via next-intl `t()`/`getTranslations()`) and resolves the wave-coupling problem: components built in plan 02-03 import keys; pages built in 02-04 import keys; layout in 02-01 already references `a11y.skipToContent`. Without this plan landing in Wave 1, any later plan's `pnpm build` would fail with "MISSING_MESSAGE" errors.

This plan also installs the **coalition-copy placeholder mechanism** (D-02): hero headline + sub-headline + entire `/agenda` body ship with the literal string `[ТЕКСТ ОТ КОАЛИЦИЯ]` so dev/staging make the missing copy visible. Plan 02-08 ships the pre-launch checklist that greps for this placeholder and fails the build if any remain when warmup is greenlit.
