## Lessons learned (for future RETROSPECTIVE.md)

**The schema-drift-vs-app-deploy split matters.** Phase 4's deploy on 2026-05-11 passed every CI gate that existed at the time — green tests, green typecheck, green build, green post-deploy smoke (`/`, `/register`, `/login`). The originally-failing surface (`/admin/*`) was not in the smoke gate. Phase 04.1's contribution is two-fold: (a) the canonical-schema-dump → delta → idempotent-backfill SQL → operator SQL Editor apply playbook becomes the bridge mechanism until `payload migrate` can run in CI; (b) the Wave 6 smoke contract proves that admin shell + the Phase 4 critical path (approve flow with `moderation_log` INSERT in the same transaction) work end-to-end on prod against the reconciled schema. Both are reusable: (a) for the next Payload schema change (Phase 3 voting when D-LawyerTrack unpauses), (b) as the minimum-acceptable deploy-confidence gate for any future change touching Payload collections.

**Operator-applied manual DDL is the bridge.** Project memory `project_payload_schema_constraint.md` documents the constraint; this phase put it through end-to-end production exercise. The `.planning/ops/2026-05-12-payload-schema-phase-04-backfill.sql` file pattern (idempotent, 2x-apply-tested, paired with `04.1-APPLY-LOG.md` operator sign-off) is the production-ready discipline for any future Payload collection addition.

**Smoke gates must exercise the originally-failing surfaces.** A green `/`, `/register`, `/login` gate said "deployment succeeded" while admin was crashing within minutes. Wave 6's expanded smoke (admin login + admin views + DB-side audit-log SQL verification + email-pipeline end-to-end + each public surface) is the contract that should be embedded in deploy.yml — at minimum as a manual-gate checklist for any future Payload-schema-touching PR, ideally as an automated post-deploy Playwright run.

**Worker handlers need contract tests, not just integration tests.** The 3 broken Phase 5 email kinds had passed all prior CI gates — the bug was symptom-only and only fired when Brevo received the malformed payload in production. Quick 260514-k4x's 3 render-level + 3 source-level grep tests now lock the regression class (any future `textContent: ''` reintroduction fails CI). The pattern (assert non-empty textContent on the rendered payload AND a source-text grep gate) should propagate to other handler families before they ship.
