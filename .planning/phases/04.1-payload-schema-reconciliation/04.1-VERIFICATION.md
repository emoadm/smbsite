1. **Plan-author UI-shape drift** (Smokes 1 + 2): plan expected Attribution + Moderation Queue links in admin side nav, and the moderation queue UI to use status-based tabs (Pending/Approved/Rejected/Hidden) with Topic column. Actual implementation: views are URL-only (no `beforeNavLinks` component shipped); queue tabs partition by submission **kind** (Предложения / Сигнали / DSA сигнали); columns are Подадено / Тип / Заглавие / Подател / Действие. Implementation is correct and consistent with what Phase 02.1 (Attribution) and Phase 4 actually shipped — the plan author's expectations had drifted from code. Follow-on todo filed for adding nav links: `.planning/todos/pending/2026-05-14-editor-nav-links-attribution-moderation.md` (`D-EditorNavLinks`, post-04.1 editor-UX polish).

2. **Plan-author route drift** (Smoke 3 Part A): plan referenced `/predlozheniya/нов` as the proposal-submission route; actual route per `src/app/(frontend)/member/predlozhi/page.tsx` is **`/member/predlozhi`** (signed-in member route). Adjusted in flight.

3. **Plan-author body-length drift** (Smoke 3 Part A): plan suggested submission body "Test от Phase 04.1 verification" (~31 chars); `src/lib/submissions/zod.ts:30` requires `min(50)`. Used a 94-char Bulgarian test body that meets the validator.

4. **Phase 5 worker textContent regression discovered as side-effect** (Smoke 3 Part D): `worker.tsx` passed `textContent: ''` to Brevo for 3 Phase 4 email kinds; Brevo 400'd with `missing_parameter`. Fixed inline via Quick Task **260514-k4x** (commits `b22669c` + `1428dc5` + `5bfe42f`, pushed 2026-05-14 ~14:30 UTC). Six regression-lock tests added to prevent reoccurrence. Phase 04.1 schema-reconciliation goal was unaffected — the DB-side audit-log contract passed on both attempts; only the async email step needed the fix.

5. **Smoke 5 empty-state expected**: `/problemi` renders the empty-state copy (no SVG) because no approved problem reports exist yet on prod. `src/app/(frontend)/problemi/page.tsx:40` gating is by design; the page returning 200 is the actual proof that queries against the reconciled `submissions` table run cleanly.

6. **Diagnostic-query timing artifact** (Smoke 3 Part C): my initial `moderation_log` SELECT used `WHERE created_at > NOW() - INTERVAL '10 minutes'` which returned 0 rows even though the row existed. Re-running by `target_id` (D3 diagnostic) confirmed presence. Likely a clock/timezone artifact in the Neon SQL Editor session; not a data issue.
