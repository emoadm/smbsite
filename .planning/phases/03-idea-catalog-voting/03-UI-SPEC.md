## Typography

Declared roles (4 sizes, 2 weights). All sizes already exist in `globals.css` `@theme` (Phase 2). Phase 3 does NOT introduce new sizes; it picks from the existing 8-size ramp.

| Role | Size | Weight | Line Height | Used For |
|------|------|--------|-------------|----------|
| Body | 16px (`--text-base` = 1rem) | 400 (Roboto regular) | 1.5 (`--leading-normal`) | Idea card excerpt, idea-detail Lexical paragraph render, vote-count copy, member dashboard prose, profile field values |
| Label | 14px (`--text-sm` = 0.875rem) | 600 (Roboto semibold) | 1.5 (`--leading-normal`) | Topic chip text, vote-count secondary line ("за членове"), profile field labels, anomaly table column headers, "Отбор на редактора" badge |
| Heading | 24px (`--text-2xl` = 1.5rem) | 800 (Gilroy ExtraBold via `--font-display`) | 1.2 (`--leading-snug`) | Idea card title, `/admin/views/vote-anomalies` page title, IdeaSidebar section titles |
| Display | 36px (`--text-4xl` = 2.25rem) | 800 (Gilroy ExtraBold via `--font-display`) | 1.15 (`--leading-tight`) | Idea-detail H1 (the idea title); `/member/profile` page H1; `/idei` page H1 ("Идеи на коалицията") |

Cyrillic-specific rules:
- **No `letter-spacing` adjustments** on Bulgarian text (Cyrillic kerning ≠ Latin kerning; tracking applies only to ALL-CAPS Latin lockups, none in Phase 3).
- **`hyphens: manual` only** for body prose (Cyrillic auto-hyphenation in browsers is unreliable for Bulgarian; rely on natural breaks). Set on `<article>` Lexical root.
- **Idea title `text-wrap: balance`** required on cards AND on detail H1 (CSS `text-wrap: balance` — supported in all evergreen browsers; prevents single-word orphans on Bulgarian titles which average ~12 characters/word and break poorly at small grid widths).
- **`font-feature-settings: "tnum"` on numeric vote counts** (the `123 одобряват` and `83% одобрение` strings) — ensures tabular figures so the count doesn't visually shift between renders.

### Accent (`--color-primary`) reserved-for list — EXHAUSTIVE

Phase 3 uses the Sinya navy accent ONLY for these elements. Any other usage is a policy violation and a checker fail.

1. **Vote button rendered-selected state** (D-24): the user's current choice — `<Button variant="default" aria-pressed="true">` — fills with `bg-primary` and `text-primary-foreground`. The opposite (un-selected) button is `variant="outline"` with `border-input` + `text-foreground`.
2. **Editor's-pick badge** (D-08, D-18): `<Badge>` with `bg-primary text-primary-foreground` + a `<Star className="size-3" />` lucide icon. Label: "Отбор на редактора". Pill shape, 10px text-xs, 4px horizontal padding, 2px vertical padding, `rounded-full`. Position: top-right of card, 8px inset from card edge.
3. **Approve-% progress bar fill** (`<Progress>` `indicatorClassName="bg-primary"`).
4. **Primary CTA buttons:** "Гласувай" alias label (where used as a section CTA on `/member` panel only — see Copywriting), "Изнеси моите данни" placeholder button, "Замрази отображаването" / "Размрази" toggle.
5. **Active page indicator on Pagination** (`<PaginationLink isActive>` → `bg-primary`).
6. **Filter-row "Нулирай филтъра" reset link** (text only, `text-primary underline-offset-4 hover:underline`).
7. **Active topic chip** (when chip is selected/applied): `bg-primary text-primary-foreground`. Inactive chip: `bg-muted text-muted-foreground border-input`.
8. **My Activity panel item current-choice indicator pill** ("Одобрено от теб" / "Не одобрено от теб"): `bg-primary text-primary-foreground`. Label-size, rounded-full.

**Explicitly NOT accent:** card hover state (use `hover:ring-1 hover:ring-secondary/50`), card focus ring (use `focus-visible:ring-2 focus-visible:ring-ring/50` — `--color-ring` is already navy at low opacity), idea-detail body links inside Lexical (use `text-primary underline-offset-4 hover:underline` — same as Phase 2 inline links; counts toward accent budget but is content-driven, not chrome).

**Filter row layout:**

**Topic chip (`<Badge variant="outline" asChild>` wrapping a `<button>`):**
- Inactive: `bg-muted text-muted-foreground border-input` + 4px x 12px padding + `rounded-full` + `text-sm font-semibold`.

- Click toggles in/out of filter set; URL updates via shallow router replace; server re-renders.

**Sort dropdown (`<DropdownMenu>` triggered by `<Button variant="outline">`):**
- Trigger button shows current sort label + `ChevronDown` icon (lucide, size-4).

- Trigger button is keyboard-focusable; menu uses Radix DropdownMenu primitive (already in `@/components/ui/dropdown-menu`).

**`<VoteCountDisplay role="public" | "member" | "frozen">`:**

  - Below, single line: `<p className="text-base">{n} одобряват  ·  {p}% одобрение</p>` — uses tabular figures, 16px Roboto regular.
  - **NO reject count.** D-02 + Pitfall 9.
- `revealed=true & role="member"`:
  - Same as public, plus a second line below: `<p className="text-sm text-muted-foreground">{rejectN} не одобряват <span className="ml-2 inline-flex rounded-full bg-warning/10 text-warning border border-warning/20 px-2 py-0.5 text-xs font-semibold">за членове</span></p>`.
  - **The "за членове" pill is mandatory** (RESEARCH Pitfall 2 — self-attributing screenshot defense). Cannot be removed by planner.
- `revealed=true & role="frozen"` (only inside Payload admin Ideas edit screen — D-23 silent freeze on public): same as member but with an additional "Замразено от %{name} на %{date}" Alert above. Public + member pages render `revealed=true & role="public" | "member"` ignoring the frozen flag (silent — D-23).

**`<VoteButtons>` (client island, mounts only for email-verified members; otherwise the slot renders an Alert "За да гласуваш, потвърди имейла си." — locked copy not in D-25; **add to bg.json**):**

Visual rules (per D-24 + RESEARCH Pattern 8):

- Layout: `<div className="flex flex-col gap-3 sm:flex-row sm:gap-4">` — stack vertically on mobile (sub-640px), side-by-side on tablet+. Each button `flex-1` so equal-width.
- Both buttons present always; current choice rendered selected:
  - **Selected (current choice):** `<Button variant="default" size="lg" aria-pressed="true" className="min-h-[44px] flex-1">` — `bg-primary text-primary-foreground`. Includes lucide icon: `Check` (size-5) for "Одобрявам", `X` (size-5) for "Не одобрявам".
  - **Unselected:** `<Button variant="outline" size="lg" aria-pressed="false" className="min-h-[44px] flex-1">` — `border-input bg-background text-foreground hover:bg-muted`. Same icon, but `text-muted-foreground`.
- Disabled-while-pending: `useTransition` per RESEARCH Pattern 8. Visual: `opacity-50 pointer-events-none`. NO spinner inside button (stays compact); a small `<Loader2 className="size-4 animate-spin">` appears in the top-right corner of the active button, replacing the icon.
- Click on CURRENT button → retract action; toast "Гласът ти е оттеглен. [Отмени]" (5-sec undo, Sonner action slot).
- Click on OTHER button → change action; toast "Гласът ти е променен." (success toast, 3-sec auto-dismiss, no action slot).
- Click on either button when `currentChoice === null` → cast action; no toast (the button visual flips to selected — toast would be redundant).

**Aria semantics (mandatory for accessibility; RESEARCH Pattern 8 + WAI-ARIA APG button-toggle):**
- `aria-pressed` is the SINGLE source of truth for selection state. Color is supportive, NOT primary.
- `aria-label` on each button INCLUDES the action consequence: `aria-label="Одобрявам — натисни за оттегляне"` when selected; `aria-label="Одобрявам"` when not.
- `aria-busy="true"` while `pending`.
- Screen reader announcement on state change: an off-screen `<div role="status" aria-live="polite">` next to the buttons updates with "Гласът ти е записан." / "Гласът ти е променен на 'Одобрявам'." / "Гласът ти е оттеглен."

**Hint copy below buttons:** `<p className="mt-4 text-sm text-muted-foreground">Гласът ти е таен. Един глас на акаунт.</p>` — locked. Reinforces D-01 anonymous + D-13 unique-per-account.

**CAPTCHA-on-suspicion (D-11) inline placement:**
- Trigger: server returns `{ ok: false, reason: 'turnstile-required' }`. The client island captures this from the Server Action result and renders `<TurnstileChallenge>` directly between the vote buttons and the hint copy (push hint copy down).
- Layout: full-width within the prose container, `my-4` vertical margin.
- Above the widget, inline `<p className="text-sm text-foreground">Моля потвърди, че не си бот.</p>` (D-25 locked copy).
- Widget itself is the standard Cloudflare Turnstile JS-rendered iframe (Phase 1 widgetId pattern).
- After successful verify, the client retries the vote silently; widget unmounts.
- Modal? **NO.** Modal would interrupt the flow. Inline matches the "subtle suspicion check" anti-abuse intent.

**Hard rate-limit error (D-11 hard threshold):**
- Server returns `{ ok: false, reason: 'hard-block' }`. Client renders `<Alert variant="destructive">` BELOW the vote buttons (replacing the hint copy temporarily for 30 sec, then auto-dismissing).
- Copy: "Превишаваш допустимата честота на гласуване. Опитай отново след няколко минути." (D-25 locked).
- Icon: `OctagonX` (lucide, size-5).
- Vote buttons remain visually present but `disabled` for 60 seconds (countdown shown via `aria-live` announcement, no visible timer to avoid frustration).

**Network error fallback** (Server Action throws):
- Client `<Alert variant="destructive">` with title "Възникна грешка." and body "Моля опитай отново." (locked — add to bg.json under `vote.error.*`).
- Vote buttons re-enabled immediately; no auto-retry.

## Accessibility Contract

Phase 3 must meet WCAG 2.1 Level AA (BRAND-04, scheduled for Phase 6 final audit but Phase 3 ships AA-compliant).

| Concern | Rule |
|---------|------|
| Color contrast | All text ≥ 4.5:1 against its background. Specifically: `--color-muted-foreground` (`#475569`) on `--color-background` (`#FFFFFF`) = 7.6:1 ✓; `--color-primary-foreground` (`#FFFFFF`) on `--color-primary` (`#004A79`) = 9.7:1 ✓; `--color-warning` (`#D97706`) on white = 4.8:1 ✓ for normal-size text; on `--color-warning/10` background = 4.5:1 (borderline — only used for label-size text 14px+ which qualifies as large under WCAG when bold). |
| Focus visible | Every interactive element shows the standard shadcn focus ring (`focus-visible:ring-2 focus-visible:ring-ring focus-visible:ring-offset-2`). NEVER rely on color-only state changes. |
| Keyboard navigation | `/idei` cards are entirely keyboard-actionable (`<Card>` wrapped in `<Link>`); Tab moves through filter chips → sort dropdown → cards in DOM order → pagination. No hidden focusable elements. |
| Vote-button aria-pressed | RESEARCH Pattern 8 mandate. Selection conveyed by `aria-pressed`, NOT by color alone. |
| Live regions | Vote state changes announced via `<div role="status" aria-live="polite">`. Cooling indicator inside `aria-live="polite"`. Toast notifications via Sonner's built-in `aria-live`. |
| Reduced motion | `prefers-reduced-motion` already handled in `globals.css` (Phase 2). Phase 3 does NOT add new animations beyond what's in shadcn primitives, which already respect the rule. |
| Screen-reader-only labels | "за членове" pill: `<span className="sr-only">Видимо само за регистрирани членове.</span>` accompanying the visible pill (provides full context to SR users). |
| Alt text on hero image | Mandatory: editor must fill `hero.alt` field in Payload Ideas collection (validated server-side; missing alt blocks publish). Empty alt allowed for decorative-only images via explicit checkbox. |
| Touch targets | All buttons ≥ 44×44 pixels (WCAG 2.5.5). Vote buttons: explicit `min-h-[44px]`. Topic chips: 32×min-64 desktop, 36×min-80 mobile. |
| Bulgarian language declared | `<html lang="bg">` already set (Phase 1). Phase 3 adds NO `lang=` overrides anywhere. |
